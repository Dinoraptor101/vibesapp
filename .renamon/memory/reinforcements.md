### cache_duplication_diagnosis
**Date:** 2025-12-12  
**Trigger:** 1,873 hits / 1,873 misses—exact 50% ratio, statistically impossible  
**Action:** Traced to `build.yml` using inconsistent `hashFiles` patterns:
- `build` job: explicit pattern `hashFiles('package-lock.json', 'apps/*/...')`  
- `e2e-tests` job: glob `hashFiles('**/package-lock.json')` → different hash!  
**Root cause:** Two jobs creating separate cache entries, each hitting own cache  
**Fix:** Unified all patterns to explicit lock file paths

**Creating GitHub Issues (preferred method):**
```bash
gh issue create --repo Dinoraptor101/vibesapp \
  --title "Issue title here" \
  --body "Markdown body here"
```

**Key insights:**
- `gh auth status` confirms authentication
- Use `--repo Dinoraptor101/vibesapp` explicitly

**Why not local API?**

**Future self:** When Partner says "create an issue," use `gh issue create`. Don't overthink it.
