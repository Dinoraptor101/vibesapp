# Project Overview

## Mission Statement

VibesApp is a **social network without the media nonsense** designed to connect real people together and foster wholesome content through community self-moderation and a hearts-based engagement system.

## Core Principles

### 🌍 Location-Based Community

- **Proximity-based interactions** - Users can set a radius (50-150km) for posts shown in their feed
- **Local focus** - Encourages real-world community building and authentic connections
- **Privacy-first** - Location data is user-controlled, with GPS and manual city input

### 🎯 Community-Driven Content Moderation

- **Hearts system** - Users can heart (like) posts to show appreciation
- **Report system** - Users can report inappropriate content for moderation
- **Admin oversight** - Dedicated admin panel for managing flagged posts and users

### 🚫 Anti-Social Media Problems

- **No advertisements** - Clean, user-focused experience
- **No bots** - Authentication via unique Pigeon ID system
- **No algorithms** - Chronological, location-filtered feed
- **No infinite scroll toxicity** - Grid-based post display with intentional navigation

## Platform Goals

### Short-term Objectives

- Build authentic local communities
- Establish effective community moderation through reporting
- Create positive engagement patterns through hearts
- Achieve sustainable user growth

### Long-term Vision

- Become the go-to platform for local community building
- Demonstrate viability of location-based social networks
- Expand globally while maintaining local focus
- Influence industry toward healthier social media models

## Target Audience

### Primary Users

- **Community-minded individuals** seeking authentic local connections
- **Young adults (18-35)** comfortable with location-based services
- **Quality-over-quantity** users who prefer thoughtful engagement
- **Privacy-conscious** individuals wanting alternatives to big tech platforms

### Use Cases

- Discovering local events and activities
- Finding like-minded people nearby (via MBTI and polarity matching)
- Sharing local recommendations and insights
- Building neighborhood communities
- Organizing local meetups and activities

## Competitive Advantage

### Technical Innovation

- **Location-proximity reactions** - Adjustable radius for content discovery
- **Personality-based matching** - MBTI and polarity profiles for better connections
- **Progressive Web App** - Offline-ready with service worker support
- **Real-time messaging** - Socket.IO-powered direct messaging and group chats

### User Experience

- **Mobile-first design** - Progressive Web App capabilities
- **Clean, distraction-free interface** - Focus on content and connections
- **Meaningful interactions** - Hearts and comments for engagement
- **Real-time features** - Instant messaging for active users

## Success Metrics

### Community Health

- **Engagement rates** - Hearts and comments per post
- **Content quality** - Low rate of reported posts
- **User retention** - Daily/monthly active users
- **Local engagement** - Interactions within geographic communities

### Platform Growth

- **User acquisition** - Organic growth through word-of-mouth
- **Geographic coverage** - Expansion to new cities/regions
- **Feature adoption** - Usage of messaging and profile features
- **Community formation** - Evidence of real-world meetups/connections

## Technology Philosophy

### Privacy by Design

- Minimal data collection (username, bio, location, MBTI, polarity)
- User control over location sharing
- No third-party tracking
- Transparent data usage

### Sustainable Development

- Clean, maintainable codebase
- Comprehensive testing strategy
- Proper QA/staging environments
- Continuous deployment practices

### User-Centric Features

- Accessibility considerations (theme options, responsive design)
- Performance optimization (PWA, efficient data loading)
- Intuitive user interface (minimal learning curve)
- Cross-platform compatibility (web-based, mobile-friendly)
